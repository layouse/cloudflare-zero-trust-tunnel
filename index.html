☁️ Cloudflare Tunnel Debian 12 / 13 için adım adım kurulum rehberi

Apache sanal konakları ile birden çok alt alan adını tek bir tünel üzerinden güvenle yayınlayın.

HTTPS Zero-Trust Cloudflare

1. Hazırlık: Apache Sanal Konaklar

Her alt alan adı için bir web dizini ve sanal konak dosyası oluşturulur. Aşağıdaki örnekte example.com ana alan adınız ve app1, app2, app3 alt alan adlarınız olsun.

# Web dizinlerini oluştur
sudo mkdir -p /var/www/html/app1 /var/www/html/app2 /var/www/html/app3

# Örnek index dosyaları
echo "<h1>App 1</h1>" | sudo tee /var/www/html/app1/index.html
echo "<h1>App 2</h1>" | sudo tee /var/www/html/app2/index.html
echo "<h1>App 3</h1>" | sudo tee /var/www/html/app3/index.html

# Sanal konak dosyalarını oluştur (her biri için)
sudo tee /etc/apache2/sites-available/app1.conf > /dev/null <<EOF
<VirtualHost *:80>
    ServerName app1.example.com
    DocumentRoot /var/www/html/app1
</VirtualHost>
EOF

sudo tee /etc/apache2/sites-available/app2.conf > /dev/null <<EOF
<VirtualHost *:80>
    ServerName app2.example.com
    DocumentRoot /var/www/html/app2
</VirtualHost>
EOF

sudo tee /etc/apache2/sites-available/app3.conf > /dev/null <<EOF
<VirtualHost *:80>
    ServerName app3.example.com
    DocumentRoot /var/www/html/app3
</VirtualHost>
EOF

# Konakları etkinleştir ve Apache’yi yeniden yükle
sudo a2ensite app1 app2 app3
sudo systemctl reload apache2
💡 Not: Kendi alan adınız ve alt alan adlarınızla değiştirin. DNS kayıtlarınızın Cloudflare üzerinden yönetildiğinden emin olun.

2. Cloudflare Depo Ekleme ve cloudflared Kurulumu

# GPG anahtarını indir
sudo mkdir -p /usr/share/keyrings
curl -L https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null

# Depoyu ekle
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared bookworm main" | sudo tee /etc/apt/sources.list.d/cloudflared.list

# Kurulum
sudo apt update && sudo apt install -y cloudflared

3. Tünel Oluşturma ve Yapılandırma

# Cloudflare hesabına giriş yap (çıkan URL'yi tarayıcıda aç)
cloudflared tunnel login

# Tüneli oluştur (örnek ad: my-tunnel)
cloudflared tunnel create my-tunnel

Oluşturma işlemi sonunda size bir tunnel ID ve credentials.json dosyasının yolu verilecektir. Bu bilgileri ~/.cloudflared/config.yml dosyasına aşağıdaki gibi yazın:

tunnel: YOUR_TUNNEL_ID
credentials-file: /home/username/.cloudflared/YOUR_TUNNEL_ID.json

ingress:
  - hostname: app1.example.com
    service: http://localhost:80
  - hostname: app2.example.com
    service: http://localhost:80
  - hostname: app3.example.com
    service: http://localhost:80
  - service: http_status:404
🔁 Önemli: YOUR_TUNNEL_ID ve kullanıcı yolunu kendi değerlerinizle değiştirin. İsterseniz daha fazla alt alan ekleyebilirsiniz.

4. DNS Rotalarını Oluşturma

Her alt alan adını tünel ile ilişkilendirmek için aşağıdaki komutları çalıştırın:

cloudflared tunnel route dns my-tunnel app1.example.com
cloudflared tunnel route dns my-tunnel app2.example.com
cloudflared tunnel route dns my-tunnel app3.example.com
✅ Bu komutlar Cloudflare DNS’e otomatik olarak CNAME kayıtları ekleyecektir.

5. cloudflared Servisini Kurma ve Başlatma

sudo cloudflared --config /home/username/.cloudflared/config.yml service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared

Servisin durumunu kontrol etmek için:

sudo systemctl status cloudflared

⚠️ Karşılaşılan Yaygın Hatalar ve Çözümleri

❌ Hata 1: Address already in use: make_sock: could not bind to port 80

Neden: Nginx gibi başka bir web sunucusu 80. portu kullanıyor.
Çözüm: 
sudo systemctl stop nginx
sudo systemctl disable nginx
sudo systemctl start apache2
❌ Hata 2: Code: 1003, An A, AAAA, or CNAME record with that host already exists

Neden: Cloudflare DNS’de aynı alt alan adına ait eski bir kayıt var.
Çözüm: Cloudflare Dashboard’dan ilgili kaydı manuel olarak silin, ardından route dns komutunu tekrar çalıştırın.
❌ Hata 3: cloudflared service is already installed / Conflicting configuration

Neden: Önceki kurulumdan kalan servis veya konfigürasyon dosyaları mevcut.
Çözüm: 
sudo cloudflared service uninstall
sudo rm -rf /etc/cloudflared
sudo cloudflared --config /home/username/.cloudflared/config.yml service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared
💡 İpucu: Tünelinizin doğru çalışıp çalışmadığını test etmek için cloudflared tunnel --config ~/.cloudflared/config.yml --url localhost:80 komutunu kullanabilirsiniz.

1. Preparation: Apache Virtual Hosts

Create web directories and virtual host configurations for each subdomain. In this example, example.com is your main domain, and app1, app2, app3 are subdomains.

# Create web directories
sudo mkdir -p /var/www/html/app1 /var/www/html/app2 /var/www/html/app3

# Sample index files
echo "<h1>App 1</h1>" | sudo tee /var/www/html/app1/index.html
echo "<h1>App 2</h1>" | sudo tee /var/www/html/app2/index.html
echo "<h1>App 3</h1>" | sudo tee /var/www/html/app3/index.html

# Virtual host configurations
sudo tee /etc/apache2/sites-available/app1.conf > /dev/null <<EOF
<VirtualHost *:80>
    ServerName app1.example.com
    DocumentRoot /var/www/html/app1
</VirtualHost>
EOF

sudo tee /etc/apache2/sites-available/app2.conf > /dev/null <<EOF
<VirtualHost *:80>
    ServerName app2.example.com
    DocumentRoot /var/www/html/app2
</VirtualHost>
EOF

sudo tee /etc/apache2/sites-available/app3.conf > /dev/null <<EOF
<VirtualHost *:80>
    ServerName app3.example.com
    DocumentRoot /var/www/html/app3
</VirtualHost>
EOF

# Enable sites and reload Apache
sudo a2ensite app1 app2 app3
sudo systemctl reload apache2
💡 Note: Replace with your own domain and subdomains. Make sure DNS is managed by Cloudflare.

2. Add Cloudflare Repository & Install cloudflared

# Download GPG key
sudo mkdir -p /usr/share/keyrings
curl -L https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null

# Add repository
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared bookworm main" | sudo tee /etc/apt/sources.list.d/cloudflared.list

# Install
sudo apt update && sudo apt install -y cloudflared

3. Create and Configure the Tunnel

# Authenticate with Cloudflare
cloudflared tunnel login

# Create a named tunnel (e.g., my-tunnel)
cloudflared tunnel create my-tunnel

You will get a tunnel ID and a path to credentials.json. Use them in ~/.cloudflared/config.yml:

tunnel: YOUR_TUNNEL_ID
credentials-file: /home/username/.cloudflared/YOUR_TUNNEL_ID.json

ingress:
  - hostname: app1.example.com
    service: http://localhost:80
  - hostname: app2.example.com
    service: http://localhost:80
  - hostname: app3.example.com
    service: http://localhost:80
  - service: http_status:404
🔁 Important: Replace YOUR_TUNNEL_ID and the user path with your own values. You can add more subdomains as needed.

4. Create DNS Routes

cloudflared tunnel route dns my-tunnel app1.example.com
cloudflared tunnel route dns my-tunnel app2.example.com
cloudflared tunnel route dns my-tunnel app3.example.com
✅ These commands automatically add CNAME records in Cloudflare DNS.

5. Install and Start cloudflared Service

sudo cloudflared --config /home/username/.cloudflared/config.yml service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared

Check service status:

sudo systemctl status cloudflared

⚠️ Common Errors & Troubleshooting

❌ Error 1: Address already in use: make_sock: could not bind to port 80

Cause: Another web server (e.g., Nginx) is using port 80.
Solution: 
sudo systemctl stop nginx
sudo systemctl disable nginx
sudo systemctl start apache2
❌ Error 2: Code: 1003, An A, AAAA, or CNAME record with that host already exists

Cause: A stale DNS record exists for the same subdomain.
Solution: Delete the old record from Cloudflare Dashboard, then rerun the route dns command.
❌ Error 3: cloudflared service is already installed / Conflicting configuration

Cause: Leftover service or config files from a previous installation.
Solution: 
sudo cloudflared service uninstall
sudo rm -rf /etc/cloudflared
sudo cloudflared --config /home/username/.cloudflared/config.yml service install
sudo systemctl start cloudflared
sudo systemctl enable cloudflared
💡 Tip: To test the tunnel without installing the service, use cloudflared tunnel --config ~/.cloudflared/config.yml --url localhost:80.
